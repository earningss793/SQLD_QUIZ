🏆 [1일차] SQL 데이터 조회(SELECT) 마스터 퀴즈 (20문항)
1부: 데이터베이스 구조와 기초 문법 (Q1~Q5)
Q1. 데이터베이스의 '테이블(Table)'을 엑셀에 비유했을 때 가장 적절한 것은?

① 데이터가 저장되는 '시트(Sheet)'

② 데이터의 속성을 나타내는 '열(Column)'

③ 실제 데이터 한 줄인 '행(Row)'

④ 계산을 수행하는 '함수(Function)'

정답: ① 해설: 테이블은 데이터가 격자 형태로 저장되는 가장 기본적인 공간으로, 엑셀의 시트와 대응됩니다. 힌트: 전체 데이터가 담긴 '그릇'을 생각해보세요.

Q2. SQL 문법에서 "어디로부터 가져올 것인가"를 지정하는 키워드는?

① SELECT

② FROM

③ WHERE

④ INTO

정답: ② 해설: SELECT는 가져올 항목을, FROM은 그 데이터가 들어있는 테이블을 지정합니다. 힌트: "가져와(SELECT) ~로부터(FROM)" 순서를 기억하세요.

Q3. SELECT * FROM USER; 쿼리에서 아스테리스크(*)의 의미로 옳은 것은?

① 데이터의 첫 번째 행만 가져온다.

② 테이블 내의 모든 컬럼(열)을 가져온다.

③ 중복된 데이터를 제외하고 가져온다.

④ 테이블의 이름만 조회한다.

정답: ② 해설: *은 '모든 것(All)'을 의미하여 해당 테이블의 모든 컬럼을 한꺼번에 조회할 때 사용합니다. 힌트: 'All'을 상징하는 기호를 떠올려 보세요.

Q4. 여러 개의 특정 컬럼(예: 이름, 나이)을 조회할 때 컬럼명 사이에 사용하는 기호는?

① 마침표 (.)

② 세미콜론 (;)

③ 콤마 (,)

④ 콜론 (:)

정답: ③ 해설: SELECT NAME, AGE FROM USER;와 같이 여러 컬럼을 구분할 때는 콤마를 사용합니다. 힌트: 나열할 때 쓰는 일반적인 문장 부호와 같습니다.

Q5. 다음 중 SQL 문장의 끝을 알리는 '마침표' 역할을 하며, 생략 시 실행되지 않는 것은?

① :

② /

③ .

④ ;

정답: ④ 해설: SQL 문장은 반드시 세미콜론(;)으로 끝나야 명령이 완료된 것으로 인식됩니다. 힌트: 문장의 마무리를 짓는 기호입니다.

2부: 별명(Alias)과 대소문자 규칙 (Q6~Q10)
Q6. 컬럼에 별명을 붙일 때 가장 권장되는(가독성이 좋은) 키워드는?

① IS

② AS

③ LIKE

④ TO

정답: ② 해설: 컬럼명 AS 별명 형식이 별명임을 명시적으로 보여주어 가장 추천되는 방식입니다. 힌트: '~로서'라는 뜻의 영어 단어를 생각해보세요.

Q7. 별명(Alias)에 '공백'이 포함된 경우(예: 회원 이름) 반드시 사용해야 하는 기호는?

① 작은따옴표 (' ')

② 큰따옴표 (" ")

③ 대괄호 ([ ])

④ 역따옴표 ( )

정답: ② 해설: 별명에 공백이나 특수문자가 있다면 큰따옴표로 감싸야 하나의 이름으로 인식합니다. 힌트: "이름표는 크게(Big name tag)" 붙이세요!

Q8. SQL의 대소문자 구별에 대한 설명 중 틀린 것은?

① SELECT와 select는 동일하게 작동한다.

② 테이블 명은 대소문자를 가리지 않는 경우가 많다.

③ 저장된 데이터 값 'HONG'과 'hong'은 같은 데이터로 취급된다.

④ 키워드(SELECT, FROM)는 대문자로 쓰는 것이 관례적으로 보기 좋다.

정답: ③ 해설: SQL 문법 키워드는 대소문자를 가리지 않지만, 테이블 안에 들어있는 실제 '데이터 값'은 대소문자를 엄격히 구별합니다. 힌트: "명령어는 관대하지만, 알맹이(데이터)는 꼼꼼하다"고 기억하세요.

Q9. SELECT NAME 이름 FROM USER;와 같이 AS 없이 별명을 붙일 수 있는가?

① 가능하다. 한 칸 띄우기만 하면 된다.

② 불가능하다. 반드시 AS를 써야 한다.

③ 오직 숫자 별명일 때만 가능하다.

④ 테이블 이름에만 가능하다.

정답: ① 해설: AS는 생략 가능하며, 띄어쓰기만으로도 별명을 부여할 수 있습니다. 힌트: 실습 예시 문법을 다시 한번 살펴보세요.

Q10. 다음 중 문법적으로 틀린 별명 부여 방식은?

① SELECT NAME AS 이름

② SELECT NAME "이 름"

③ SELECT NAME AS '이름'

④ SELECT NAME 별명

정답: ③ 해설: 별명에는 작은따옴표(')를 쓰지 않습니다. 작은따옴표는 데이터 값을 감쌀 때 사용합니다. 힌트: 작은따옴표는 '내용물' 전용입니다.

3부: 산술 연산과 NULL (Q11~Q15)
Q11. SQL에서 사칙연산 우선순위가 가장 낮은 것은?

① * (곱하기)

② / (나누기)

③ + (더하기)

④ ( ) (괄호)

정답: ③ 해설: 곱하기와 나누기가 먼저 계산되고, 더하기와 빼기는 나중에 계산됩니다. 힌트: 초등 수학 산술 순위와 같습니다.

Q12. SELECT 100 + 200 * 2 FROM DUAL; 의 결과값은?

① 600

② 500

③ 400

④ 300

정답: ② 해설: 200 * 2가 먼저 계산되어 400이 되고, 여기에 100을 더해 500이 됩니다. 힌트: 곱셈이 먼저입니다!

Q13. 데이터베이스에서 NULL의 의미로 가장 적절한 것은?

① 숫자 0

② 공백 문자 (' ')

③ 알 수 없는 값 (Unknown / Missing)

④ 테이블의 끝 (End of Table)

정답: ③ 해설: NULL은 0도 공백도 아닌, 값이 존재하지 않거나 무엇인지 모르는 상태를 뜻합니다. 힌트: "정해지지 않은 무(無)의 상태"라고 이해하세요.

Q14. SELECT POINT * 10 FROM USER; 실행 시, POINT가 NULL인 행의 결과값은?

① 0

② 10

③ NULL

④ 에러 발생

정답: ③ 해설: NULL과 어떤 값을 산술 연산해도 그 결과는 무조건 NULL이 됩니다. 힌트: "NULL이랑 놀면 다 망한다(NULL이 된다)"는 명언(?)을 기억하세요.

Q15. 연산 순서를 강제로 바꾸고 싶을 때 사용하는 기호는?

① [ ]

② { }

③ ( )

④ < >

정답: ③ 해설: 괄호를 사용하면 괄호 안의 연산이 가장 먼저 수행됩니다. 힌트: 수학에서의 괄호 역할과 동일합니다.

4부: 심화 및 필수 테크닉 (Q16~Q20)
Q16. 오라클(Oracle)에서 테이블 데이터 없이 단순 계산(예: SELECT 1+1)을 하고 싶을 때 빌려 쓰는 가상 테이블은?

① DUMMY

② VIRTUAL

③ DUAL

④ PRACTICE

정답: ③ 해설: DUAL 테이블은 결과가 1행인 가상의 연습장 테이블로, 오라클에서 필수적인 문법 요건을 맞추기 위해 사용합니다. 힌트: "이중적인(Dual)" 느낌의 짧은 이름을 떠올리세요.

Q17. 특정 컬럼의 중복된 값을 제거하고 '종류'만 보고 싶을 때 SELECT 바로 뒤에 붙이는 키워드는?

① UNIQUE

② DISTINCT

③ SINGLE

④ GROUP

정답: ② 해설: SELECT DISTINCT GRADE FROM USER;라고 쓰면 중복을 제거한 고유한 값만 출력됩니다. 힌트: "뚜렷한, 별개의"라는 뜻의 영어 단어입니다.

Q18. 문자열 데이터(예: 이름이 'Hong'인 경우)를 감쌀 때 반드시 써야 하는 기호는?

① 작은따옴표 (' ')

② 큰따옴표 (" ")

③ 역따옴표 ( )

④ 따옴표 없음

정답: ① 해설: 데이터 내용물(문자열, 날짜 등)은 작은따옴표로 감싸야 합니다. 힌트: "내용은 작게(Small content)"!

Q19. 다음 중 SQL 문법에 대한 설명으로 옳은 것은?

① SELECT * FROM USER (세미콜론 없음)은 완벽한 문장이다.

② SELECT DISTINCT NAME, AGE는 NAME만 중복 제거한다.

③ SELECT 10 / 0 FROM DUAL은 결과로 NULL을 낸다.

④ SELECT NAME AS "회원 이름"은 정상적인 문법이다.

정답: ④ 해설: 별명에 공백이 있을 때 큰따옴표를 쓴 ④번이 정석입니다. ①은 세미콜론 누락, ②는 나열된 컬럼 조합 전체를 중복 제거함, ③은 0으로 나누기 에러가 발생합니다. 힌트: 오늘 배운 '큰따옴표'의 용도를 다시 확인하세요.

Q20. SELECT (POINT + 100) * 2 AS "최종 점수" FROM USER; 쿼리의 해석으로 옳은 것은?

① 점수에 100을 더한 뒤 2를 곱하고, 그 컬럼 이름을 "최종 점수"라고 표시한다.

② 점수에 2를 곱한 뒤 100을 더하고, "최종 점수"라는 테이블에서 가져온다.

③ 모든 유저의 POINT를 2로 바꾼다.

④ POINT가 100인 사람만 찾아서 2를 곱한다.

정답: ① 해설: 괄호 안의 덧셈이 먼저 수행되고 그다음 곱셈이 진행되며, AS 뒤의 별명이 결과창에 나타납니다. 힌트: 산술 연산의 순서와 별명의 위치를 복합적으로 생각해보세요.
