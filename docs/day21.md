

### 📝 SQLD 1과목+2과목 40문항

#### [PART 1] 데이터 모델링의 이해 (1과목 - 32문항)

**1. 데이터 모델링의 3대 특징에 대한 설명으로 가장 부적절한 것은?**
① 추상화: 현실 세계의 불필요한 디테일을 버리고 일정한 형식으로 간략하게 표현하는 것이다.
② 단순화: 누구나 쉽게 이해할 수 있도록 약속된 규약이나 표기법을 사용하여 표현하는 것이다.
③ 명확화: 대상에 대한 애매모호함을 제거하고 한 가지 의미로 해석되도록 기술하는 것이다.
④ 구체화: 현실 세계의 모든 업무 프로세스와 데이터를 하나도 빠짐없이 1:1로 완벽하게 복제하여 구현하는 것이다.

* **정답:** ④
* **해설:** 데이터 모델링은 현실을 완벽하게 복제하는 것이 아니라, 필요한 부분만 뽑아내는 '추상화' 과정이 핵심입니다.
* **힌트:** 모델링의 3대 특징은 '추.단.명'입니다.

**2. 데이터 모델링의 진행 3단계 중 '논리적 데이터 모델링'에 대한 설명으로 옳은 것은?**
① 특정 DBMS(Oracle, MySQL 등)의 물리적인 특성과 성능을 가장 우선적으로 고려한다.
② 식별자, 속성, 관계 등을 정확히 표현하며, 데이터의 중복을 제거하기 위해 정규화(Normalization)를 수행한다.
③ 전사적 관점에서 주제 영역을 정의하고 핵심 엔터티를 도출하여 추상화 수준이 가장 높은 ERD를 작성한다.
④ 인덱스(Index)와 파티셔닝(Partitioning) 등 실제 하드웨어 저장 구조를 설계한다.

* **정답:** ②
* **해설:** ①, ④는 물리적 모델링, ③은 개념적 모델링에 대한 설명입니다. 정규화는 논리적 모델링의 핵심 활동입니다.
* **힌트:** "개논물 난다" 중 중간 단계로, 재사용성이 가장 높고 뼈대를 튼튼하게 잡는 단계입니다.

**3. 모델링의 3가지 관점 중 '주문이 발생하면 상품의 재고가 줄어든다'와 같이 데이터와 업무 흐름 사이의 연관성을 분석하는 관점은?**
① 데이터(Data) 관점
② 프로세스(Process) 관점
③ 상관관계(Correlation) 관점
④ 무결성(Integrity) 관점

* **정답:** ③
* **해설:** 데이터와 프로세스가 서로 어떻게 영향을 주고받는지(CRUD 매트릭스 활용 등)를 분석하는 것은 상관관계 관점입니다.
* **힌트:** 상호 작용을 확인하는 관점입니다.

**4. ANSI-SPARC 3단계 스키마 구조 중, 개별 사용자나 응용 프로그램 관점에서 데이터베이스의 일부분만을 정의하며 하나의 DB에 여러 개가 존재할 수 있는 것은?**
① 개념 스키마
② 내부 스키마
③ 외부 스키마
④ 물리 스키마

* **정답:** ③
* **해설:** 사용자가 접근하는 뷰(View)의 집합인 외부 스키마에 대한 설명입니다.
* **힌트:** 유저가 눈으로 보는 '마이페이지' 화면을 떠올려보세요.

**5. 다음 중 '물리적 데이터 독립성'에 대한 설명으로 가장 알맞은 것은?**
① 개념 스키마가 변경되어도 외부 스키마에는 영향을 주지 않는 성질이다.
② 응용 프로그램 코드가 변경되어도 사용자는 이를 알지 못하는 현상이다.
③ 저장 장치를 교체하거나 인덱스를 추가하는 등 내부 스키마가 변경되어도 개념 스키마나 외부 스키마에 영향을 주지 않는 성질이다.
④ 외부 스키마와 개념 스키마 사이를 연결해 주는 '논리적 사상(Mapping)'을 의미한다.

* **정답:** ③
* **해설:** ①은 논리적 데이터 독립성에 대한 설명입니다. 내부 구조의 변화가 논리적 구조에 영향을 주지 않는 것이 물리적 독립성입니다.
* **힌트:** 하드디스크를 SSD로 바꾼다고 쿼리를 새로 짤 필요는 없습니다.

**6. 엔터티(Entity)가 성립하기 위한 6가지 필수 조건에 포함되지 않는 것은?**
① 해당 업무에서 반드시 필요하고 관리하고자 하는 정보여야 한다.
② 반드시 1개 이상의 인스턴스(데이터 행)를 가져야 한다.
③ 반드시 2개 이상의 속성(컬럼)을 가져야 한다.
④ 다른 엔터티와 최소 1개 이상의 관계가 성립해야 한다. (통계/코드성 제외)

* **정답:** ②
* **해설:** 엔터티는 반드시 '2개 이상'의 인스턴스 집합이어야 합니다. 데이터가 1개뿐이라면 테이블로 구축할 의미가 없습니다.
* **힌트:** 집합은 최소 2개부터 시작합니다.

**7. 다음 중 '발생 시점'에 따른 엔터티의 분류로 알맞게 짝지어진 것은?**
① 유형 엔터티, 개념 엔터티, 사건 엔터티
② 외부 엔터티, 내부 엔터티, 교차 엔터티
③ 기본 엔터티, 중심 엔터티, 행위 엔터티
④ 마스터 엔터티, 트랜잭션 엔터티, 이력 엔터티

* **정답:** ③
* **해설:** 유무형에 따른 분류는 [유·개·사], 발생 시점에 따른 분류는 [기·중·행]입니다.
* **힌트:** 원래 있던 것인지, 그로 인해 파생되었는지를 따지는 분류입니다.

**8. 물리적 형태는 없으나 업무상 관리해야 할 추상적 개념인 '개념 엔터티'에 해당하는 것은?**
① 사원
② 강사
③ 주문내역
④ 보험상품

* **정답:** ④
* **해설:** 사원, 강사는 유형 엔터티, 주문내역은 사건/행위 엔터티입니다. 보험상품, 조직, 부서 등은 개념 엔터티입니다.
* **힌트:** 눈에 보이지 않고 만질 수 없는 대상을 고르세요.

**9. 속성(Attribute)에 대한 설명으로 올바르지 않은 것은?**
① 업무에서 필요로 하는 인스턴스의 더 이상 쪼갤 수 없는 최소의 데이터 단위이다.
② 하나의 속성은 반드시 1개의 속성값(단일값)만 가져야 하며 이를 원자성이라 한다.
③ 정해진 주식별자에 함수적으로 완전 종속되어야 한다.
④ 한 개의 엔터티는 1개의 속성만 가질 수도 있다.

* **정답:** ④
* **해설:** 엔터티 필수 조건에 따라, 한 개의 엔터티는 반드시 2개 이상의 속성을 가져야 합니다.
* **힌트:** 이름만 달랑 있는 테이블은 없습니다.

**10. 속성의 분류 중 '설계 속성(Designed Attribute)'에 해당하는 것은?**
① 고객의 생년월일
② 상품의 단가
③ 직원의 사원번호
④ 이번 달 총매출액

* **정답:** ③
* **해설:** 생년월일, 단가는 기본 속성, 총매출액은 파생 속성입니다. 사원번호, 주문번호 등은 식별을 위해 인위적으로 부여한 설계 속성입니다.
* **힌트:** 원래 자연계에 존재하던 정보가 아니라, 시스템 관리를 위해 억지로 만든 번호를 찾으세요.

**11. 다음 중 '파생 속성(Derived Attribute)'의 특징으로 가장 적절한 것은?**
① 데이터 조회 시 계산을 피할 수 있어 시스템 성능을 향상시키므로 최대한 많이 생성해야 한다.
② 다른 속성으로부터 계산, 변형되어 만들어지며 데이터 정합성 유지를 위해 가급적 적게 사용하는 것이 좋다.
③ 업무로부터 추출된 가장 일반적인 속성으로, 전체 속성의 대부분을 차지한다.
④ 인스턴스를 유일하게 식별하기 위해 인위적으로 부여하는 코드성 속성이다.

* **정답:** ②
* **해설:** 파생 속성은 조회 성능에는 유리할 수 있으나, 원본 데이터가 변경될 때 함께 갱신되지 않으면 데이터 불일치(정합성 파괴)가 발생하므로 사용을 최소화해야 합니다.
* **힌트:** 재고량(파생)은 입고량과 출고량(원본)이 바뀔 때마다 다시 계산해야 하는 귀찮은 속성입니다.

**12. 의미를 더 이상 쪼갤 수 없는지(분해 여부)에 따른 분류 중, '주소(시+구+동)'에 해당하는 속성은?**
① 단일 속성
② 다중값 속성
③ 복합 속성
④ 기본 속성

* **정답:** ③
* **해설:** 여러 의미가 결합되어 있어 쪼갤 수 있는 속성을 복합 속성이라고 합니다.
* **힌트:** 의미가 복합적으로 섞여 있습니다.

**13. 한 고객이 두 개의 이메일 주소를 가지고 있을 때, 관계형 데이터베이스 모델링 관점에서 이를 해결하는 가장 올바른 방법은?**
① 이메일 컬럼 하나에 쉼표로 구분하여 두 주소를 모두 입력한다.
② 해당 고객의 인스턴스(행)를 2개 생성하여 각각 입력한다.
③ 이메일을 다중값 속성으로 보고, 별도의 엔터티로 분리(1차 정규화)한다.
④ 두 번째 이메일은 무시하고 첫 번째 이메일만 저장한다.

* **정답:** ③
* **해설:** 하나의 속성에 여러 값이 들어가는 '다중값 속성'은 원자성 원칙에 위배되므로 반드시 별도 엔터티로 빼내어 1:N 관계로 연결해야 합니다.
* **힌트:** 관계형 DB는 하나의 셀에 여러 값을 구겨 넣는 것을 극도로 싫어합니다.

**14. 각 속성이 가질 수 있는 값의 범위(데이터 타입, 크기, 제약조건 등)를 의미하는 데이터 모델링 용어는?**
① 인스턴스

④ 뷰

* **힌트:** 물리적 모델링 단계에서 VARCHAR2(10)처럼 구현되는 것입니다.

**15. 다음 중 데이터 모델링 용어들의 상호 포함 관계로 올바르게 나열된 것은?**
① 1개의 엔터티는 1개의 인스턴스를, 1개의 인스턴스는 다수의 속성을 갖는다.
② 1개의 엔터티는 2개 이상의 인스턴스를, 1개의 속성은 여러 개의 속성값을 갖는다.
③ 1개의 엔터티는 2개 이상의 인스턴스를, 1개의 속성은 오직 1개의 속성값만 갖는다.
④ 1개의 속성은 2개 이상의 엔터티를, 1개의 인스턴스는 오직 1개의 속성값만 갖는다.

* **정답:** ③
* **해설:** 엔터티/인스턴스와 속성은 '2개 이상', 속성과 속성값은 '원자성(1개)' 원칙을 따릅니다.
* **힌트:** 3교시에서 외웠던 포함 관계 공식(2 이상, 2 이상, 오직 1)입니다.

**16. 다음 `<보기>` 중 엔터티 명명 규칙으로 옳은 것을 모두 고르시오.**

**17. 다음 `<보기>` 중 3단계 스키마 구조와 데이터 독립성에 대한 설명으로 올바른 것을 모두 고르시오.**

> **<보기>**
> a. 논리적 데이터 독립성은 개념 스키마가 변경되어도 외부 스키마에 영향을 주지 않는 것을 말한다.
> b. 물리적 사상(Mapping)은 개념 스키마와 내부 스키마를 연결하며, 물리적 데이터 독립성을 보장한다.
> c. 개념 스키마는 조직 전체 관점의 통합된 설계도로 DB에 오직 1개만 존재한다.
> d. 외부 스키마가 변경되면 개념 스키마도 반드시 연쇄적으로 수정되어야 한다.

① a, b, c
② a, b, d
③ b, c, d
④ a, b, c, d

* **정답:** ① (a, b, c)

* **힌트:** 뼈대(개념)가 중심이고, 껍데기(외부)는 여러 개 씌울 수 있습니다.

**33. 다음 중 SQL 문장의 논리적 실행 순서로 올바른 것은?**
① FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
② SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY

**34. 마케팅 비용 분석 중 아래 SQL을 실행했을 때의 결과 추론으로 알맞은 것은?**
[데이터: A캠페인 비용=1000, B캠페인 비용=NULL, C캠페인 비용=2000]

```sql
SELECT SUM(COST) + SUM(NULLIF(COST, 1000)) FROM CAMPAIGN;

```

* **보기:**
① 3000
② 5000
③ NULL
④ 에러 발생
* **정답:** ②
* **해설:** `SUM(COST)`는 NULL을 무시하고 1000+2000 = 3000입니다. `NULLIF(COST, 1000)`은 1000과 같으면 NULL을 반환하므로 A캠페인은 NULL, B도 NULL, C는 2000이 되어 SUM 결과는 2000입니다. 3000 + 2000 = 5000.
* **힌트:** `NULLIF(A, B)`는 A와 B가 같으면 NULL을 반환합니다.

**35. 마케팅 캠페인별 평균 ROAS를 구하고자 합니다. 이때 평균 ROAS가 300% 이상인 그룹만 조회하기 위한 올바른 SQL 구문은?**
① `SELECT CAMPAIGN_ID, AVG(ROAS) FROM AD_DATA WHERE AVG(ROAS) >= 300 GROUP BY CAMPAIGN_ID;`
② `SELECT CAMPAIGN_ID, AVG(ROAS) FROM AD_DATA GROUP BY CAMPAIGN_ID HAVING AVG(ROAS) >= 300;`
③ `SELECT CAMPAIGN_ID, AVG(ROAS) FROM AD_DATA GROUP BY CAMPAIGN_ID WHERE ROAS >= 300;`
④ `SELECT CAMPAIGN_ID, AVG(ROAS) FROM AD_DATA HAVING AVG(ROAS) >= 300;`

* **정답:** ②
* **해설:** 집계 함수(AVG)를 조건으로 필터링하려면 GROUP BY 이후에 HAVING 절을 사용해야 합니다. WHERE 절에는 집계 함수를 쓸 수 없습니다.
* **힌트:** 사과를 박스(Group)에 담은 후 무게를 재는(Having) 과정입니다.

**37. 다음 데이터를 보고 윈도우 함수의 결과를 예측하시오.**
[결제액(AMT): 100, 100, 300 (시간순 정렬)]

```sql
SELECT SUM(AMT) OVER(ORDER BY TIME ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)

```

② 200, 200, 500

**38. 다음 SQL의 실행 결과 건수로 옳은 것은?**
[회원 테이블 M1: {A, B, C}], [탈퇴자 테이블 M2: {B, NULL}]

```sql
SELECT * FROM M1 
WHERE MEMBER_ID NOT IN (SELECT MEMBER_ID FROM M2);

```

① 2건
② 1건
③ 0건
④ 3건

* **정답:** ③

* **힌트:** NOT IN과 NULL이 만나면 블랙홀이 됩니다.

**39. 다음 중 삭제 명령어 `TRUNCATE`와 `DELETE`에 대한 설명으로 옳은 것은?**
① `DELETE`는 DDL 명령어이므로 실행 즉시 Auto-Commit 된다.
② `TRUNCATE`는 데이터를 지울 때 복구용 로그를 남기지 않으므로 `DELETE`보다 속도가 압도적으로 빠르다.
③ `TRUNCATE`는 WHERE 절을 사용하여 특정 행만 선택적으로 삭제할 수 있다.
④ `DELETE`는 실행 후 롤백(Rollback)이 절대 불가능하다.

* **정답:** ②
* **해설:** `TRUNCATE`는 저장 공간 수위표를 초기화하는 DDL로 로그를 남기지 않아 매우 빠르고 롤백이 불가합니다. 반면 `DELETE`는 DML로 롤백이 가능하며 WHERE 절 사용이 가능합니다.
* **힌트:** TRUNCATE는 방을 완전히 폭파시키는 것이고, DELETE는 짐을 하나씩 빼는 것입니다.

**40. 다음 `<보기>` 중 SQL 기본 문법 및 활용에 대해 올바른 것을 모두 고르시오.**

> **<보기>**
> a. 오라클에서 DDL(CREATE, ALTER 등)을 실행하는 순간 그전의 DML 내역은 자동으로 COMMIT 된다.
> b. UNION 연산자는 중복을 허용하며 내부적으로 정렬(Sort)을 수행하지 않아 속도가 빠르다.
> c. 서브쿼리 내에서는 원칙적으로 ORDER BY 절을 사용할 수 없다. (인라인 뷰 제외)
> d. ROWNUM 절은 서브쿼리를 통한 데이터 정렬 전에 번호가 먼저 매겨지므로 Top-N 쿼리 작성 시 주의해야 한다.

① a, b, c
② a, c, d
③ b, c, d
④ a, b, c, d

* **정답:** ② (a, c, d)
* **해설:** b는 틀렸습니다. 중복을 허용하고 정렬하지 않아 빠른 것은 `UNION ALL`입니다. `UNION`은 중복을 제거하고 내부 정렬을 수행하여 속도가 느립니다.
* **힌트:** 합집합에서 All이 붙어야 그냥 막 갖다 붙이는 것입니다.
